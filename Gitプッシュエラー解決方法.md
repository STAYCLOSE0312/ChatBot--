# 🔧 Gitプッシュエラー解決方法

## ⚠️ エラーの原因

「参照仕様をリモートにプッシュできません。最初に"Pull"を実行して変更を統合してください。」

このエラーは、GitHub（リモート）にローカルにない変更があるため、プッシュする前にプル（pull）する必要があることを示しています。

---

## ✅ 解決方法

### 方法1: CursorのUIから操作（簡単）

1. **エラーダイアログを閉じる**（「キャンセル」をクリック）

2. **ソース管理パネル**を開く（`Ctrl + Shift + G`）

3. **「変更を同期」ボタンの横にある「...」（3点リーダー）**をクリック

4. **「プル、プッシュ」**または**「プル」**を選択

5. プルが完了したら、再度**「変更を同期」**をクリック

### 方法2: ターミナルから操作（確実）

1. **ターミナル**を開く（`Ctrl + `` `）

2. 以下のコマンドを順番に実行：

```bash
# ステップ1: リモートの変更を取得
git pull origin main
```

**競合が発生した場合**:
- メッセージが表示されたら、そのままEnterキーを押す
- または、競合を解決する必要がある場合があります

```bash
# ステップ2: 変更をプッシュ
git push origin main
```

---

## 🔄 手順の詳細

### ステップ1: プル（Pull）

リモートリポジトリ（GitHub）の変更をローカルに取得します。

**CursorのUIから**:
1. ソース管理パネルで「変更を同期」ボタンの横の「...」をクリック
2. 「プル、プッシュ」を選択

**ターミナルから**:
```bash
git pull origin main
```

### ステップ2: プッシュ（Push）

ローカルの変更をリモートリポジトリ（GitHub）にアップロードします。

**CursorのUIから**:
1. 「変更を同期」ボタンをクリック

**ターミナルから**:
```bash
git push origin main
```

---

## ⚠️ 競合が発生した場合

プル時に競合（コンフリクト）が発生した場合：

1. **競合ファイルを確認**
   - 競合が発生したファイルが表示されます

2. **競合を解決**
   - ファイルを開いて、競合箇所を修正
   - `<<<<<<<` と `>>>>>>>` の間の内容を確認して修正

3. **変更を保存**

4. **再度コミット**
   ```bash
   git add .
   git commit -m "競合を解決"
   ```

5. **プッシュ**
   ```bash
   git push origin main
   ```

---

## 💡 補足情報

### なぜこのエラーが発生するのか？

- GitHubのWebサイトから直接ファイルを編集した場合
- 別のPCやブラウザから変更を加えた場合
- 複数の人が同じリポジトリを編集している場合

### 今後の予防策

プッシュする前に、常にプルを実行する習慣をつけましょう：

```bash
git pull origin main
git push origin main
```

または、CursorのUIで「変更を同期」ボタンを使うと、自動的にプルとプッシュの両方が実行されます。
